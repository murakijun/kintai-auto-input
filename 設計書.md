## 1. システム概要

Outlookの送信済みメール（始業時報告・終業時報告）を自動解析し、勤務表Excelファイルへ始業時間・終業時間・勤務形態を自動入力するPythonスクリプトです。

### 目的

- 勤務表への手動入力作業を自動化し、記入漏れ・ミスを防ぐ
- 月次の勤務表作成にかかる時間を削減する

## 4. 処理フロー

```
[スクリプト起動]
      │
      ▼
[config.ini 読み込み]
  ・対象年月
  ・送信者メールアドレス
  ・Excelファイルパス
  ・プレビューモード
      │
      ▼
[Excelファイル決定]
  config.ini に path あり → そのまま使用
  path なし or ファイル未存在 → ファイル選択ダイアログを表示
                                 → 選択パスを config.ini に自動保存
      │
      ▼
[Outlook接続]
  送信済みアイテムフォルダを開く
      │
      ▼
[メール取得・フィルタリング]
  条件①：件名に「始業時報告」または「終業時報告」を含む
  条件②：送信日が対象年月に一致する
  条件③：送信者アドレスが config.ini の sender_email と一致する ← 他者メール除外
      │
      ├── [始業時報告メール]
      │     └── 始業時間・勤務形態を抽出 → 当日データに格納
      │
      └── [終業時報告メール]
            └── 終業時間を抽出 → 当日データに格納
                始業時報告が未取得の場合は終業メールの始業時間で補完
      │
      ▼
[Excel書き込み]
  日付ごとに対応する行を特定し、セルに値を書き込む
  プレビューモード時は書き込まず表示のみ
      │
      ▼
[完了メッセージ表示]
```

## 5. 設定ファイル仕様（config.ini）

```ini
[user]
sender_email = yourname@example.com   # 自分のメールアドレス

[target]
year  = 2026    # 処理対象年
month = 4       # 処理対象月

[excel]
path =          # Excelファイルのフルパス（空の場合ダイアログ表示）

[outlook]
folder_type = 5  # 5=送信済み / 6=受信トレイ

[options]
preview_only = false  # true=確認のみ / false=実際に書き込む
```

## 6. メール解析仕様

### 6-1. 始業時報告

| 抽出対象 | メール内の記述例 | 正規表現パターン |
|---------|----------------|-----------------|
| 始業時間 | `始業時間：09:00` | `始業時間[：:]\s*(\d{1,2}):(\d{2})` |
| 勤務形態 | `在宅/出社勤務：在宅勤務` | `在宅[/／]出社勤務[：:]\s*(.+?)[\n\r]` |

### 6-2. 終業時報告

| 抽出対象 | メール内の記述例 | 正規表現パターン |
|---------|----------------|-----------------|
| 終業時間 | `終業時間：17:30` | `終業時間[：:]\s*(\d{1,2}):(\d{2})` |
| 始業時間（補完用） | `始業時間：09:00` | `始業時間[：:]\s*(\d{1,2}):(\d{2})` |

### 6-3. 勤務形態マッピング

| メール記述 | Excelの備考欄（P列）|
|-----------|---------------------|
| 在宅勤務 | `在宅勤務(客先業務)` |
| 出社勤務 | （空欄） |

## 7. Excel書き込み仕様

### 対象ファイル

`フレックス版_社員勤務報告書_169村木純_YYYYMM_ver.X.XX.xlsx`

### 対象シート

`勤務表`

### 書き込みセル対応表

| 日付 | 行番号 | 書き込む列 | 内容 |
|------|--------|-----------|------|
| 4月1日 | 6行目 | C列（始業時間） | `timedelta`形式 例: `timedelta(seconds=32400)` = 09:00 |
| 4月1日 | 6行目 | D列（終業時間） | `timedelta`形式 例: `timedelta(seconds=63000)` = 17:30 |
| 4月1日 | 6行目 | P列（備考） | 文字列 例: `在宅勤務(客先業務)` |
| 4月N日 | N+5行目 | 同上 | 同上 |

> **行番号の計算式:** `行 = 5 + 日付の日`（1日→6行目、2日→7行目、…、30日→35行目）

### 時刻の型変換

Excelの時刻セルは `timedelta` として保存されます。

```
"09:00"  →  timedelta(seconds=32400)   (= 9時間 × 3600秒)
"17:30"  →  timedelta(seconds=63000)   (= 17.5時間 × 3600秒)
```

## 8. 送信者フィルタ仕様

他の人が同様のフォーマットでメールを送信している場合に、自分のメールのみを処理するフィルタです。

### 判定ロジック

```
1. config.ini の sender_email が設定されている
        → item.SenderEmailAddress と大文字小文字を無視して比較
        → 一致する場合のみ処理

2. sender_email が未設定
        → Outlook の CurrentUser.Address から自動取得して使用
        → 取得できない場合はフィルタなし（全件処理）
```

### 処理されないメールの例

- 同じ部署の他のメンバーが「終業時報告」を送信してきた場合
- 転送されて受信トレイに届いた勤務報告メール

## 9. エラーハンドリング

| エラー状況 | 対処 |
|-----------|------|
| pywin32 未インストール | エラーメッセージを表示してインストール手順を案内 |
| Excelファイルが見つからない | ファイル選択ダイアログを表示 |
| Excelシートが存在しない | エラーメッセージと利用可能シート一覧を表示して終了 |
| メールが見つからない | 警告を表示して正常終了（Excelは変更しない） |
| メール本文の解析失敗 | 該当メールをスキップして処理継続 |
| 他者のメールを検出 | スキップしてカウントを表示 |

## 11. 月次運用手順

1. **月が変わったら:** `config.ini` の `year` / `month` を更新する
2. **初回確認:** `preview_only = true` で実行し、内容を目視確認する
3. **本実行:** `preview_only = false` に変更して再実行する
4. **Excelを開いて確認:** 入力内容に問題がないか確認して提出する
